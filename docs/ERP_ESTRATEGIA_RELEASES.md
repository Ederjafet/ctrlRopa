# ERP - Estrategia de releases

## Politica propuesta

El proyecto debe pasar de cambios directos a releases controlados.

## Fases por release

1. Analisis.
2. Documentacion.
3. Identificacion de impacto.
4. Implementacion pequena.
5. QA.
6. Regresion.
7. Actualizacion documental.
8. Empaque.
9. Validacion en QA.
10. Liberacion.

## Requisitos antes de liberar

- Repositorio Git confirmado.
- Version/tag.
- Checklist QA.
- Migraciones aplicadas en ambiente copia.
- Backup de base.
- Usuario QA por rol.
- Logs backend/frontend activos.
