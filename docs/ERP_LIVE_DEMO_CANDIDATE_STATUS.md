# ERP LIVE - Estado demo candidate

Fecha: 2026-05-19  
Fase: LIVE-Q - usability hardening

## Decision

Estado tecnico: `GO tecnico pendiente de validacion final`.

Estado UX demo: `GO condicionado`.

## Lo que esta listo para demo

- Layout responsive por dispositivo.
- Product Spotlight.
- Consola de operador.
- Activity feed humano.
- Metricas demo.
- i18n ES/EN.
- Microcopy comercial inicial.
- Safe area local para Android/tablet en LIVE.

## Lo que no se debe prometer

- Integracion real Facebook/TikTok/Instagram/YouTube.
- Realtime backend.
- WebSockets.
- Ventas live finales.
- Pagos live automatizados.
- IA productiva.
- Reportes live finales.

## Bloqueos para produccion real

- Falta smoke visual en dispositivos fisicos.
- Falta modelo real de eventos.
- Falta backend realtime o polling controlado.
- Falta permisos especificos por rol LIVE.
- Falta auditoria de acciones LIVE reales.

## Recomendacion

Usar como demo comercial controlada despues de validar en tablet real. No usar como promesa de funcionalidad realtime productiva.
