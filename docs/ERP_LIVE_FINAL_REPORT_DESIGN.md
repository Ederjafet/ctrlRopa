# ERP LIVE-L - Final Report Design

Fecha: 2026-05-19

## Objetivo

Definir el reporte final deseado para una sesion `En vivo`, sin modificar reportes runtime existentes en esta fase.

## Resumen del En vivo

Campos sugeridos:

- company,
- sucursal,
- fecha/hora inicio,
- fecha/hora cierre,
- duracion,
- operador,
- presentadora,
- plataforma externa,
- estado final.

## Resultados comerciales

- productos mostrados,
- productos reservados,
- productos vendidos/cobrados,
- monto estimado,
- monto cobrado,
- monto pendiente,
- productos no vendidos con interes.

## Clientes

- clientes nuevos,
- clientes recurrentes,
- compradores,
- interesados sin compra,
- espectadores mas activos.

## Interacciones

- comentarios totales,
- reacciones totales,
- comentarios por producto,
- interesados por producto,
- conversion interes -> reserva.

## Inventario y recomendaciones

- prendas mas solicitadas,
- tallas mas pedidas,
- marcas con mas interes,
- productos para repetir en otro canal,
- mercancia sugerida para otra transmision,
- prendas que deben rematarse o moverse a otro flujo.

## Uso operacional

El reporte debe servir para:

- cierre de transmision,
- seguimiento comercial,
- compra futura de mercancia,
- medicion de presentadoras/operadores,
- auditoria de reservas/cobros.

## Permisos sugeridos

- Supervisor/dueno: ver completo.
- Operador: ver resumen operativo.
- Reportes: ver analitica.
- Soporte HPSQ-SOFT: solo con permiso auditado.

## Fuera de alcance actual

- Crear endpoint nuevo.
- Crear reporte runtime.
- Cambiar reportes existentes.
- Integrar plataformas externas.
