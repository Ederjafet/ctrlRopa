# ERP LIVE - Flujo rapido del operador

Fecha: 2026-05-20  
Fase: LIVE-W

## Objetivo

Ayudar al operador a capturar reservas rapido sin cargar mentalmente todas las alternativas al mismo tiempo.

## Decisiones

- Cliente existente y cliente nuevo se separan visualmente.
- Prenda existente por codigo/QR es la ruta principal.
- Buscar y escanear QR quedan como ayuda secundaria.
- Alta rapida queda como excepcion operativa.
- Reserva conserva el CTA `Reservar ahora`.

## Resultado esperado

Menos botones compitiendo, menos saturacion en mobile y tablet, y mayor claridad durante transmisiones con ritmo alto.
