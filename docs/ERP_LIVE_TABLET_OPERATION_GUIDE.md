# ERP LIVE - Guia de operacion tablet

Fecha: 2026-05-19  
Fase: LIVE-P - runtime realistic flow

## Objetivo

Usar tablet como consola principal de operacion durante una transmision.

## Prioridad en tablet

1. Registrar reserva rapidamente.
2. Mantener visible producto actual.
3. Consultar reservas recientes.
4. Leer actividad reciente.
5. Revisar metricas compactas.

## Uso recomendado

### Antes de iniciar

- Iniciar sesion con usuario de sucursal.
- Abrir `En vivo`.
- Crear o seleccionar transmision.
- Validar que cliente y prendas cargan.

### Durante la transmision

- La presentadora muestra la prenda.
- El operador selecciona o escanea prenda.
- El operador selecciona cliente.
- El operador registra precio y reserva.
- Supervisor observa pulso y reservas.

### Despues de la transmision

- Revisar reservas recientes.
- Cobrar pendientes.
- Cerrar transmision.
- Documentar incidencias o mejoras.

## Buenas practicas UX

- Mantener tablet horizontal.
- Evitar abrir muchas pantallas externas durante la transmision.
- Usar busqueda por prenda si el codigo/QR no esta a la mano.
- No cerrar la transmision hasta confirmar que no quedan reservas pendientes de captura.

## Limitaciones actuales

- No hay sincronizacion realtime con plataformas externas.
- No hay roles LIVE separados en backend.
- La actividad visual contiene datos simulados controlados.
