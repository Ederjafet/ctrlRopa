# ERP LIVE - Flujo compacto del operador

Fecha: 2026-05-21  
Fase: LIVE-W refinamiento adicional

## Objetivo

Reducir la sensacion de formulario ERP en la consola En vivo y acercarla a una operacion tactil rapida durante transmision.

## Principios

- Menos texto explicativo.
- Menos botones con la misma prioridad.
- Acciones relacionadas agrupadas.
- Inputs y CTA principales mas dominantes.
- Secundarios visibles pero discretos.

## Aplicacion actual

- Cliente existente y cliente nuevo quedan separados.
- Codigo/QR domina la captura de prenda.
- Buscar y escanear quedan como asistencia.
- Crear cliente/prenda queda como excepcion.
- El bloque superior de transmision se mantiene unido como contexto operacional.

## Validacion pendiente

- Mobile Android: revisar que no haya scroll excesivo.
- Tablet landscape: revisar que captura quede visible sin bajar demasiado.
- Desktop: revisar que no queden huecos laterales cuando widgets estan ocultos.
