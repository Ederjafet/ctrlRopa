# ERP LIVE-N - Device Experience Strategy

Fecha: 2026-05-19

## Objetivo

Definir como debe sentirse `En vivo` por dispositivo para evitar una pantalla unica comprimida.

## Desktop

Debe sentirse:

- comercial,
- ejecutivo,
- analitico,
- apto para supervision.

Prioridades:

- metricas,
- actividad,
- productos destacados,
- reservas,
- contexto de transmision.

## Tablet

Debe sentirse:

- consola operativa principal,
- tactil,
- rapida,
- clara para operador/capturista.

Prioridades:

1. reservas recientes,
2. registrar reserva,
3. producto actual,
4. transmision actual,
5. metricas compactas.

Reglas:

- maximo 2 columnas,
- evitar tarjetas gigantes,
- esconder ayudas largas,
- resumir eventos,
- mantener CTA visible.

## Mobile

Debe sentirse:

- rapido,
- simple,
- de apoyo.

Mostrar primero:

- transmision actual,
- producto,
- registrar reserva,
- reservas recientes.

Ocultar o reducir:

- analytics complejos,
- eventos largos,
- metricas secundarias.

## QA UX

Validar:

- texto no se parte verticalmente,
- botones tactiles comodos,
- tarjetas con jerarquia clara,
- sin scroll excesivo inicial en tablet,
- sin terminos tecnicos visibles en espanol,
- modales de cliente/prenda legibles.

## Decision

LIVE debe evolucionar como producto live-commerce, no como modulo administrativo del ERP.
