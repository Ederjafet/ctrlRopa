# Demo LIVE SaaS - Presentation Script

Fecha: 2026-05-18  
Rama: `feature/demo-a-live-saas-presentation`

## Objetivo de la Presentacion

Mostrar el avance del ERP SaaS multi-compania de HPSQ-SOFT y posicionar LIVE commerce como diferenciador comercial.

La demo debe demostrar:

- ERP SaaS multiempresa en una sola plataforma.
- Aislamiento por empresa para datos operativos.
- Control de clientes, inventario y lotes.
- LIVE commerce con UX mejorada.
- Multi idioma ES/EN.
- Metricas demo de engagement para explicar la vision futura.
- Roadmap realista sin prometer funciones no terminadas.

## Audiencia Objetivo

- Clientes potenciales.
- Comercios con venta en vivo, boutique, ropa, bazar o lavanderia.
- Equipo interno HPSQ-SOFT.
- QA y soporte.
- Socios comerciales.
- Tomadores de decision que necesitan entender valor operativo y escalabilidad SaaS.

## Mensaje Principal

HPSQ-SOFT esta construyendo una plataforma ERP SaaS para controlar operacion, inventario, lotes y ventas asistidas, con una vision clara hacia LIVE commerce y analitica de engagement.

La promesa de valor no es solo "capturar datos"; es operar con orden, trazabilidad, multiempresa y una experiencia preparada para venta moderna.

## Guion Demo Paso a Paso

### 1. Apertura

Que decir:

"Vamos a mostrar una version en evolucion del ERP HPSQ-SOFT. La base ya trabaja con arquitectura multi-compania, y el modulo LIVE muestra hacia donde vamos: vender, apartar, medir interaccion y operar desde una misma plataforma."

Valor:

- Producto con vision SaaS.
- Diferenciador por LIVE commerce.
- Enfoque enterprise: estabilidad, QA y roadmap.

### 2. Login

Accion:

- Abrir frontend.
- Iniciar sesion con usuario QA/demo.

Que decir:

"El acceso se controla por usuario, rol, sucursal y tenant activo. En fases recientes se preparo la sesion tenant-aware para que cada empresa opere dentro de su propio contexto."

Beneficio:

- Seguridad por empresa.
- Control de acceso.
- Base para clientes SaaS en un solo servidor.

### 3. Dashboard

Accion:

- Mostrar dashboard.

Que decir:

"El dashboard concentra pendientes operativos: inventario, lotes, pagos, reservas y actividad de la sucursal. La meta es que el operador vea primero que necesita atender."

Beneficio:

- Operacion diaria mas clara.
- Menos busqueda manual.
- Mejor seguimiento.

### 4. Customers Tenant-Aware

Accion:

- Abrir clientes.
- Mostrar que clientes pertenecen a la empresa activa.

Que decir:

"Clientes ya fue una de las primeras tablas operativas tenant-aware. Esto significa que una empresa no debe ver clientes de otra."

Beneficio SaaS:

- Aislamiento de datos.
- Confianza para operar varias empresas en una sola plataforma.

### 5. Items / Inventario Tenant-Aware

Accion:

- Abrir inventario/items.
- Mostrar busqueda o listado.

Que decir:

"Inventario tambien esta preparado por tenant. Codigos y QR pueden existir en distintas empresas sin mezclarse, siempre bajo validacion backend."

Beneficio:

- Control de prendas.
- Menos errores de inventario.
- Preparado para crecimiento multiempresa.

### 6. Batches / Lotes Tenant-Aware

Accion:

- Abrir lotes.
- Mostrar folio/lote.

Que decir:

"Lotes tambien ya avanzaron a tenant-aware. Esto permite controlar recepcion, clasificacion y trazabilidad sin exponer lotes entre empresas."

Beneficio:

- Trazabilidad desde recepcion.
- Control de calidad.
- Mejor operacion de inventario.

### 7. Empresa A/B Aislamiento

Accion:

- Explicar dataset QA Empresa A/B.
- Si el ambiente esta preparado, mostrar que A no ve datos de B.

Que decir:

"Para probar seguridad SaaS no basta con decirlo. Creamos Empresa A y Empresa B con datos duplicados para validar que cada una ve solo lo suyo."

Beneficio:

- Pruebas reales de aislamiento.
- Menor riesgo de fuga de datos.
- Base para SaaS multiempresa.

### 8. LIVE

Accion:

- Abrir LIVE.

Que decir:

"LIVE es el modulo que conecta operacion con comercio en vivo. Hoy permite organizar sesiones live, capturar reservas y preparar una experiencia visual de engagement."

Beneficio:

- Venta asistida.
- Captura durante transmisiones o eventos.
- Flujo preparado para evolucionar a integraciones externas.

### 9. Selector ES/EN

Accion:

- Cambiar idioma ES/EN.

Que decir:

"LIVE ya esta preparado con base multi idioma. Esto permite pensar en clientes con operacion bilingue o demos comerciales en ingles."

Beneficio:

- Mejor presentacion comercial.
- Base para internacionalizacion futura.

### 10. Estado Operativo LIVE

Accion:

- Mostrar tarjeta de estado: sin live, abierto, activo o cerrado.

Que decir:

"El operador necesita saber en que estado esta la sesion. La pantalla ahora sugiere el siguiente paso y reduce acciones ambiguas."

Beneficio:

- Menos errores operativos.
- Mejor experiencia para vendedor/cajero.
- QA mas facil de validar.

### 11. Metricas Demo

Accion:

- Mostrar panel de metricas demo.

Que decir:

"Estas metricas son simuladas para demo. Sirven para mostrar la direccion del producto: viewers, comentarios, reacciones, engagement y productos destacados."

Beneficio:

- Vision clara de LIVE commerce.
- Narrativa comercial fuerte.
- Base visual para futuras metricas reales.

### 12. Engagement

Accion:

- Mostrar viewers, engagement %, comentarios y reacciones.

Que decir:

"La meta es que el negocio no solo venda, sino que entienda que productos generan interes y en que momento ocurre la mayor interaccion."

Beneficio:

- Mejor decision comercial.
- Mejor seleccion de productos.
- Analitica post-live futura.

### 13. Productos Destacados

Accion:

- Mostrar productos destacados demo.

Que decir:

"En el futuro, el vendedor podra destacar productos durante el live, medir clics y relacionar esa actividad con reservas o ventas."

Beneficio:

- LIVE mas ordenado.
- Mejor conversion.
- Menos improvisacion.

### 14. Timeline de Eventos

Accion:

- Mostrar timeline demo.

Que decir:

"El timeline nos ayuda a contar lo que paso en el live: inicio, producto destacado, comentario, reaccion y cierre."

Beneficio:

- Trazabilidad.
- Auditoria operativa.
- Base para reportes post-live.

### 15. Cierre

Que decir:

"Lo que mostramos hoy combina base ERP, seguridad multiempresa y una vision LIVE commerce. Lo importante es que el crecimiento esta siendo controlado: primero estabilidad y aislamiento, despues metricas reales, integracion Facebook y flujos comerciales mas avanzados."
